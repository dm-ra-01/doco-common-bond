# About-Us Documentation Audit

**Date:** 2026-03-11\
**Scope:** `documentation/common-bond` — `docs/about-us/` section\
**Auditor:** Ryan Ammendolea\
**Standard:** Internal documentation completeness; audience: advisors, future
hires, partner relationships

---

## Executive Summary

The Docusaurus internal corporate site hosts rich strategy, compliance, and
technical documentation, but has no section that provides a cohesive,
human-readable narrative about _who Common Bond is_. The `docs/about-us/`
directory was created but left empty. This audit identifies **8 documentation
and configuration gap findings**, all rated 🟡 Low — no security or process risk
exists, but the missing section represents a material gap for external
onboarding and internal coherence. Implementation will create 8 content pages, a
sidebar entry, a navbar item, and a cross-link from the homepage.

## 1. About-Us Documentation Gap

### 1.1 Content Completeness

**Strengths:**

- The `docs/strategy/strategy-vision/` section contains deep, well-structured
  documentation on the founder, vision, core values, competitive strengths, and
  why-now analysis — providing a rich source of truth to draw from.
- `strategy/brand/common-bond.md` and `operations/eos/vto.md` collectively
  document the company identity and operational cadence.
- `strategy/strategy-vision/founder.md` contains a comprehensive professional
  profile with education, clinical career, engineering career, service history,
  and personal vision statement.

**Gaps:**

- **INFO-01** `docs/about-us/` — Empty directory; no index, no sub-pages. A
  visitor encounters an empty generated index. Missing narrative on who Common
  Bond is, who Ryan is (in a personal rather than CV style), what Receptor does,
  where we currently stand, and what we are aiming for. Requires 7 new markdown
  files.

- **INFO-04** `docs/about-us/our-technology.md` — No non-technical technology
  overview exists in the about-us section. Advisors and partners need a
  plain-language summary of the technology stack (Supabase, Cloudflare,
  OR-Tools, Python match-backend) without reading the engineering docs.

- **INFO-05** `docs/about-us/founder.md` — No chronological narrative exists for
  the Common Bond origin story. A Mermaid timeline (2005–2026) would surface the
  full journey from engineering career through clinical training to founding
  Receptor and provide credibility and context.

- **INFO-06** `docs/about-us/who-we-are.md` — The legal entity table should
  include ASIC ACN numbers alongside ABN. Confirmed: MyJMO Pty Ltd ACN 648 051
  852; Common Bond Pty Ltd ACN 694 840 394. These are public ASIC records.

- **INFO-07** All about-us pages — No `description:` frontmatter fields planned.
  Without them, Docusaurus generated-index cards show no description text and
  meta tags are empty.

### 1.2 Navigation & Discoverability

**Gaps:**

- **INFO-02** `sidebars.ts` — No `aboutUsSidebar` autogenerated entry. The
  `docs/about-us/` directory will not appear in Docusaurus navigation.

- **INFO-03** `docusaurus.config.ts` — No navbar item for About Us. Section is
  unreachable from global navigation.

- **INFO-08** `docs/intro.md` — The site homepage has no link to About Us. An
  advisor entering the site via the homepage cannot discover who Common Bond is
  without independently navigating the sidebar or searching.

---

## 2. Cross-Cutting Observations

1. **Narrative-Strategy Gap:** The site contains significant
   structured/quantitative planning materials (V/TO, revenue projections, SWOT,
   scope matrices) but no _narrative_ layer that tells the Common Bond story in
   prose. The about-us section fills this gap.

2. **Founder Page Duplication Risk:** `strategy/strategy-vision/founder.md` is a
   comprehensive CV. The new `about-us/founder.md` must be narrative-first and
   link _to_ the CV page rather than duplicating it. These are distinct
   documents serving different audiences.

3. **Legal Entity Clarity:** `who-we-are.md` is an opportunity to clearly
   document the MyJMO Pty Ltd / Common Bond Pty Ltd / Ammendolea Family Trust
   structure with full ASIC identifiers, currently undocumented anywhere on the
   site.
