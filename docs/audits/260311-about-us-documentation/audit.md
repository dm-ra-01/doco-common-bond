# About-Us Documentation Audit

**Date:** 2026-03-11\
**Scope:** `documentation/common-bond` — `docs/about-us/` section\
**Auditor:** Ryan Ammendolea\
**Standard:** Internal documentation completeness; audience: advisors, future
hires, partner relationships

---

## Executive Summary

The Docusaurus internal corporate site hosts rich strategy, compliance, and
technical documentation but has no section providing a cohesive human-readable
narrative about _who Common Bond is_. The `docs/about-us/` directory was created
but left empty. This audit identifies **13 documentation and configuration gap
findings**, all rated 🟡 Low — no security or process risk, but the missing
section represents a material gap for external onboarding and internal
coherence. Implementation will create 9 content pages, a sidebar entry, a navbar
item, and a homepage cross-link.

## 1. About-Us Core Content Gap

### 1.1 Missing Pages

**Strengths:**

- `docs/strategy/strategy-vision/` contains deep structured documentation
  (founder CV, vision, VTO, core values, scope, why-now, strengths) — rich
  source of truth.
- `strategy/brand/common-bond.md` and `operations/eos/vto.md` document corporate
  identity and operational cadence.
- The legal entity structure (MyJMO Pty Ltd / Common Bond Pty Ltd / Ammendolea
  Family Trust) is internally understood.

**Gaps:**

- **INFO-01** `docs/about-us/` — Empty directory. Requires 7 core files:
  `_category_.json`, `index.md`, `founder.md`, `who-we-are.md`, `what-we-do.md`,
  `where-we-are.md`, `our-aims.md`.
- **INFO-04** `docs/about-us/our-technology.md` — No internal technology brief
  for pre-customer engagement. All repos are private; this is an internal-only
  document.
- **INFO-09** `docs/about-us/who-we-serve.md` — No page profiling target
  customers (Medical Workforce Units) or naming Monash Health as the active
  pilot engagement.

## 2. Content Enrichment Gaps

**Gaps:**

- **INFO-05** `docs/about-us/founder.md` — No chronological origin story. A
  Mermaid timeline (2005–2026) surfaces the full
  engineering-to-clinical-to-founder journey.
- **INFO-06** `docs/about-us/who-we-are.md` — Legal entity table lacks ASIC ACN
  numbers (MyJMO Pty Ltd ACN 648 051 852; Common Bond Pty Ltd ACN 694 840 394).
- **INFO-07** All about-us pages — Missing `description:` frontmatter for
  Docusaurus SEO and generated-index card previews.
- **INFO-10** `docs/about-us/who-we-are.md` — Four brand pillars stated without
  proof. Concrete "in action" examples per pillar needed.
- **INFO-11** `docs/about-us/our-technology.md` — Full internal repository table
  (all 7 repos with one-line descriptions) needed for pre-customer technical
  conversations.
- **INFO-12** All about-us pages — No `last_reviewed:` frontmatter date.

## 3. Navigation & Discoverability Gaps

**Gaps:**

- **INFO-02** `sidebars.ts` — No `aboutUsSidebar` autogenerated entry.
- **INFO-03** `docusaurus.config.ts` — No navbar item for About Us.
- **INFO-08** `docs/intro.md` — Homepage has no link to About Us, making the
  section undiscoverable from the site entry point.
- **INFO-13** `docs/about-us/_category_.json` — Sidebar category will collapse
  by default; `collapsed: false` needed for better UX.

---

## 4. Cross-Cutting Observations

1. **Narrative-Strategy Gap:** Rich structured planning documents exist; no
   narrative layer tells the Common Bond story in prose for a non-technical
   audience.
2. **Founder Page Architecture:** `strategy/strategy-vision/founder.md` (CV) and
   `about-us/founder.md` (narrative) serve distinct audiences. Cross-link but do
   not duplicate.
3. **PMCV Status:** Internal documentation must accurately reflect PMCV as a
   _target relationship being developed_, not an established one. Monash Health
   work has generated interest but no formal agreement exists.
