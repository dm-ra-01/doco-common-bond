# ISMS Roles & Responsibilities

## 1. Overview

Information security is everyone's responsibility. As a small team, we wear
multiple hats, but distinct accountability is crucial for ISO 27001 compliance.

## 2. Key Roles

### Top Management (Founder / CEO) — Ryan Ammendolea

**Accountability:** Ultimate responsibility for the ISMS.\
**Contact:** ryan@commonbond.au **Responsibilities:**

- Approving the Information Security Policy.
- Ensuring resources are available for security initiatives.
- Conducting Management Reviews.
- Accepting residual risks.

### Information Security Manager (ISM) / CTO — Ryan Ammendolea

**Accountability:** Operational design and maintenance of the ISMS.\
**Contact:** ryan@commonbond.au | Security incidents: alert@commonbond.au | +61
403 551 813

::: note[Dual-Role Arrangement]

At this stage of the business, the Founder/CEO and ISM/CTO roles are held by the
same individual, Ryan Ammendolea. This dual-role arrangement is a documented and

It will be reviewed as the team grows.

**Responsibilities:**

- Leading the risk assessment process.
- Monitoring security metrics and incidents.
- Coordinating internal audits.
- Ensuring technical controls (e.g., Supabase RLS, Cloudflare WAF) are
  effective.

### All Employees & Contractors

**Responsibility:** "If you see something, say something." **Responsibilities:**

- Adhering to the Acceptable Use Policy.
- Reporting security incidents or weaknesses immediately.
- Completing security awareness training.
- Keeping devices secure (updates, disk encryption, strong passwords).

## 3. Contact Information

For any security concerns, contact: **security@commonbond.au** or
**alert@commonbond.au** (urgent incidents).
